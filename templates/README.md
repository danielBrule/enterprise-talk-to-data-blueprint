Templates may be extracted here later when they are useful as standalone reusable files.

Potential future templates:

* Framing decision log
* Priority question inventory
* Answerability matrix
* Metric implementation card
* Governed asset register
* Orchestration design checklist
* Golden question evaluation set
* MVP evidence pack
* Security and evaluation validation checklist
* Pilot feedback register

* Operating monitoring register

The repository is intentionally lightweight at v1. The main assets are the master blueprint, phase guides and annex packs.

# `prototype/README.md`

# Prototype

This folder is reserved for a future Talk-to-Data reference prototype.

The current repository publishes the delivery blueprint and supporting phase documents. A future prototype may demonstrate selected patterns from the blueprint, such as:

* Metadata retrieval and grounding
* Query generation and validation
* Governed metric cards
* Safe answer generation
* Logging and feedback capture
* Evaluation against golden questions
* Basic observability and evidence capture

The prototype should remain clearly separated from the methodology documents and should not include confidential data, credentials or organisation-specific assets.

Phases 6 to 9 will further define how the MVP should be validated, piloted, productionised and operated before the prototype is positioned as more than a reference implementation.
